# Plan: SCALP-Only Authority Alignment

## Background

Current state:
- `simulation/authority.py` has 3 modes (SWING/SCALP/AGGRESSIVE_SCALP)
- `generate_labels_bulk()` uses a utility function matching the engine's decision logic
- Action agreement is 81-83% vs simulation engine across all 3 modes
- The ~18% gap is from: (a) cost model (flat 8bps vs notional-based fee+slippage+funding), (b) path metrics (MFE/MAE tracking not identical), (c) same-candle ambiguity, (d) no-trade quality

User directive: **SCALP ONLY** — strip SWING and AGGRESSIVE_SCALP; all other modes are dead code for this work.

## Key Findings

1. **`total_cost_r()` IS Numba-compatible** — all functions in `simulation/engine/costs.py` (`fee_cost_r`, `slippage_cost_r`, `compute_entry_risk`) are pure scalar arithmetic with no Python object dependencies. Verified by exploration agent.

2. **Current cost model in bulk generator**: `cost_r = 0.0008 * entry / (atr * stop_mult)` (flat round-trip fee in R-units)
   **Engine fee cost**: `cost_r = entry * 0.0004 * 2 / (atr * stop_mult) = 0.0008 * entry / (atr * stop_mult)` — **IDENTICAL**
   **Engine additional slippage**: `0.0002 * (entry + atr) / (atr * stop_mult)` — not included in bulk, accounts for ~25% of cost gap

3. **80% threshold was set post-hoc** — confirmed: the test file `simulation/tests/golden/test_authority.py` is untracked/never committed. The threshold started as 1e-6 (per-row assert, failed), then 90% assert (failed), then 90% warning (still warned), then 80% warning (passes). User is correct: eşik sonradan ayarlandı.

## Plan: 3-Step Approach

### Step 1: Port full cost model to Numba

Replace the flat `round_trip_cost / entry_risk` in `_bulk_label` with the engine's formula:

```python
# In _bulk_label Numba function, inside the per-row loop:
# Instead of:
#   cost_r = round_trip_cost / entry_risk
# Do:
def _cost_in_r(entry_price, atr_val, stop_mult, taker_bps, slippage_bps):
    risk = atr_val * stop_mult
    if risk <= 0: return 0.0
    fee_r = entry_price * (taker_bps / 10000.0) * 2 / risk
    # Slippage with volatility adjustment
    adj_slip = (slippage_bps / 10000.0) * (1.0 + atr_val / entry_price)
    slip_r = entry_price * adj_slip * 2 / risk
    return fee_r + slip_r
```

This is pure scalar arithmetic — confirmed Numba-compatible. `slippage_bps = 1.0`, `taker_bps = 4.0`.

**Files to change:**
- `simulation/authority.py` — replace `cost_r = round_trip_cost / entry_risk` with full `_cost_in_r()`

Remove SWING and AGGRESSIVE_SCALP entries from `AUTHORITY_PROFILES` in `simulation/authority.py`. Keep only SCALP.

**Files to change:**
