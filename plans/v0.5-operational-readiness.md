## T1: Generic Retry Utility

**Files:**
- Create: `runtime/services/retry.py`
- Create: `runtime/tests/test_retry.py`

**Interfaces:**
- `retry_with_backoff(fn, max_retries=3, base_delay=1.0, backoff_factor=2.0, max_delay=60.0, retryable_exceptions=(Exception,)) -> Any`
- `async_retry_with_backoff(async_fn, ...) -> Awaitable[Any]`
- `is_retryable_error(exception, retryable_exceptions) -> bool`

**Behavior:**
- Exponential backoff: delay = base_delay * backoff_factor^attempt
- Jitter: add random 0-100ms to each delay
- Max delay cap at max_delay
- Only retry on retryable_exceptions
- Log each retry attempt with attempt number and exception message
- Raise the last exception after exhausting retries
- async variant supports async functions

## T6: Health Check Enhancements

**Files:**
- Modify: `runtime/api/routes/health.py`
- Create: `runtime/services/health_service.py`

**Interfaces:**
- `GET /api/v3/health/liveness` — lightweight: is the process alive?
- `GET /api/v3/health/readiness` — medium: can the app serve requests? (DB check)
- Enhanced `GET /api/v3/health` with component-level breakdown

**Behavior:**
- Liveness: always returns 200 if process is running
- Readiness: checks DB connection, returns 200/503
- Health: existing endpoint with enhanced component breakdown
- Component status: db, exchange, circuit_breaker, alerts, scan_control, credentials
